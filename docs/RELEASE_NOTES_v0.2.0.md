# v0.2.0 发布说明

## 发布定位

`v0.2.0` 是 SLZ-DevOps 从“可部署观测底座”推进到“单机自建平台最小可交付版”的首个正式里程碑。

本版本重点不在单一组件增强，而在于把平台控制面、管理面和运维面真正串成一个可运行闭环。

## 本版本新增能力

### 1. 独立前端控制台

- 新增独立前端目录 `apps/console`
- 支持作为 `console` profile 单独启用
- 与 Grafana 分离，承载平台管理界面和业务控制面

### 2. 事件中心

- 支持人工事件创建
- 支持事件确认与关闭
- 支持 Alertmanager Webhook 事件接入
- 支持事件状态持久化

### 3. CMDB 管理

- 支持服务列表查看与筛选
- 支持服务新增、编辑、删除
- 支持依赖校验和删除保护

### 4. 作业中心

- 新增 `job-runner` 服务
- 支持受控模板化作业执行
- 支持作业状态、结果摘要和日志回显
- 支持执行记录持久化

### 5. 统一平台业务库

- 事件中心、CMDB、作业中心统一落到共享 SQLite 数据库
- 默认数据库路径为 `data/platform/platform.db`

### 6. 审计与备份

- 关键平台操作写入统一审计日志
- 新增数据库备份与恢复脚本
- 新增 systemd 定时备份能力

### 7. systemd 标准化托管

- 平台服务改为模板化 systemd 安装
- 去除旧的硬编码路径依赖
- 新增 `slz-platform-backup.timer` 每日自动备份

## 默认访问入口

- 控制台: `http://<host>:14000`
- Grafana: `http://<host>:13000`
- Prometheus: `http://<host>:19090`
- Alertmanager: `http://<host>:19093`
- AI Engine: `http://<host>:18080/healthz`
- Job Runner: `http://<host>:18084`（服务接口，通常由控制台代理访问）

## 已知边界

- 当前为单机自建平台最小版，尚未引入独立业务数据库服务
- 当前控制台代理仍基于 Nginx，尚未引入统一 API Gateway/BFF
- 当前作业中心仍为受控模板模式，未开放脚本库、批量执行和审批流

## 下一阶段建议

- 引入统一 API Gateway/BFF
- 为平台业务库增加迁移与异地备份策略
- 为作业中心增加审计增强、批量执行与回滚模板
- 为事件中心增加抑制、升级与通知编排能力
